  1. Decode upload: convert the clean file to mono 16 kHz WAV.
  2. RIR convolution: optionally apply mild or severe room response from the configured RIR index.
  3. DEMAND noise: optionally mix environmental noise at the selected SNR bucket.
  4. Level effects: apply gain shift and optional clipping before channel simulation.
  5. Telephone channel: resample and band-limit to narrowband or wideband, then run an ffmpeg codec round-trip.
  6. Network impairment: optionally apply waveform dropout as a packet-loss approximation.
  7. Model rate: resample degraded audio back to 16 kHz and apply peak safety normalization.
  8. Target rule: narrowband outputs get a bandwidth-aligned clean target; wideband outputs keep the 16 kHz clean target.

RIR

Room impulse response convolution simulates reverberation before the channel.

Noise

Background noise is mixed to a target SNR from the configured noise index.

Narrowband

Uses 8 kHz channel audio and a 300-3400 Hz band limit.

Wideband

Uses 16 kHz channel audio and a 50-7000 Hz band limit.

Codec

ffmpeg performs a real encode/decode round-trip for G.711, GSM, AMR, or Opus.

Dropout

Short waveform spans are zeroed to approximate bursty packet loss.

JSONL-style metadata records the parameters, seed, file paths, channel, codec, and normalization used for the generated pair.
